# Source-first: clone the docs repo instead of crawling, when one exists

Most documentation generators embed a link back to the repository that hosts the docs source — MkDocs/Material and Docusaurus render an "Edit this page" link, Sphinx/Read-the-Docs an "Edit on GitHub". When that repo is public, the markdown source it holds is strictly better than crawled HTML: no JS rendering, no extraction loss, no rate limiting, and the original nav/structure intact. So before crawling, getdocs fetches the seed page once, scans it for a repo link (`source.detect_repo`), and — if found — clones the repo and writes an `mkdocs.yml` to serve it locally, skipping the crawl entirely.

This is **default-on** and **files-mode only**. JSONL is a page stream with no place for a clone, and `--resume` continues an existing crawl, so both bypass the check. The detector trusts only links onto known Git hosts (GitHub, GitLab, Bitbucket, Codeberg) and ranks an explicit "edit this page" affordance above a bare footer mention, because a footer "star us on GitHub" link often points at a *different* repo than the docs.

The check costs one extra fetch of the seed page even when it falls back to crawling (the decision must be made before the crawl starts, and Scrapy's fetch happens after). That is the price of the feature; `--no-clone-source` opts out for callers who only want the crawler — and crawl-mechanics tests that count seed-page fetches pass it to isolate their measurement. When detection finds nothing, the clone fails, or git is missing, getdocs silently falls back to crawling: source-first is an optimization, never a hard dependency.
